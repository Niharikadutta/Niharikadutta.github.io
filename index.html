TARS

A SELF-NAVIGATING ROBOT

Overview

TARS is a mini-robot controlled by a raspberry pi, that navigates using either manual control or artificial intelligence.

It has a 5MP camera, that it uses to take pictures, which are stored, and then at the end of the path, is compiled to form a video of the shortest path or the complete path taken by the bot.

Features of TARS

Self-Navigation

On this mode, the bot searches for the shortest path to the given destination, using A-star search algorithm. It looks for obstacles in all directions, at each new point in its path, using its ultrasonic sensor and avoids them when they are within a certain threshold defined.

All the parameters of the bot's movement are configurable: This includes the starting point, the destination, the speed with which the bot moves, the heauristics that the bot uses to gauge the shortest path and the display parameters.

GRAPHICAL USER INTERFACE

This is a graphical interface/application to TARS, which is an easygui based menu driven interface that allows run-time configuration of variables, like the speed with which TARS moves, the starting and ending points, etc. It also has a radar which tracks the bot movement and renders it on client display. Once the parameters have been set, TARS can be deployed can be run from the application and its navigational path will be displayed.

This includes a visual display of the path that TARS is sketching out in its search for the destination. The Blue colour shows all the paths/blocks traversed by TARS The Green colour shows the obstacles it faced on its way The Yellow colour shows the final shortest path it takes to the destination from the origin

CLIENT-SERVER ARCHITECTURE

In order to have a clean and robust design, and lower the use of computational resources of the Raspberry pi, we have employed a Client-Server architecture for TARS. The Raspberry Pi on TARS acts as the server while a client can be any device on the network capable of running it. Both communicate via socket programming. On starting the application, the client sends requests to the server, regarding the next step to be taken (move, look for obstacle, take a picture), and the Server on performing the task sends the appropriate response.

The main computational tasks of the algorithm are taken care of at the client, with only the instructions related to the bot's movement, obstacles, and taking pictures being communicated to the server. This has greatly reduced the load of computation on the Raspberry pi, which has limited memory of 512 MB only.

ANALYSIS OF VISUAL INPUTS

TARS has a 5 megapixel camera, which is used to take pictures at regular intervals, during the robot's navigation, which are then analyzed. For example, on the AI mode, when TARS reaches the destination, the client compiles all the pictures, taken at points on the shortest path, and combines them to form a video representation of the path traversed by the robot.

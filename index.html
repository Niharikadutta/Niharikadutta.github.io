TARS

A Self-driving bot

Overview

TARS is a robot, currently under development by two Computer Science gradutes for research as well as recreational purposes. The prime purpose of the bot is navigation which is either self-driven via A.I. algorithms or manual.

The bot is capable of taking visual feedback from the surroundings using a camera, and perform analysis on the images using Machine Learning as well as other image based algorithms.

The TARS software supports user friendly GUI, allowing the user to customize a wide array of variable to control the behavior of the bot as well as the GUI itself. We have also implemented a radar into the software which allows the user to track the bot remotely.

TARS is also capable of avoiding obstacles using an ultrasonic sensor and is powered by a power bank making it completely mobile.

FEATURES

Artificial Intelligence

We have coded several known A.I. search algorithms like A* and Breadth First Search et cetera. As TARS navigates, it may encounter obstacles which is where these algorithms come in. At each node, these algorithms help TARS to intelligently and patience compute the next steps in its path.

TARS plots its surroundings on a square graph upon which the A.I. algorithms are deployed. Every step signifies a new node and the distance between each node is customizable. If a node is blocked by an obstacle, TARS doesn't give up on it, rather it tries to reach that node from the other three sides before ruling it out as a possible option.

We are currently researching the efficiency of different flavors of heauristics in A* search algorithm under various scenarios to find optimal coefficients for fastest destination search. Our software makes the task easier as the coefficients are easily customizable via the GUI menus.
For example - Manhattan distance heuristic : f(n) = α*g(n) + β*h(n)
where α and β are varied.

GRAPHICAL USER INTERFACE

For a simple and user friendly experience, TARS software has an easygui based menu driven interface. The main menu allows user to connect the client application to the Raspberry Pi, configure runtime varibles to control the algorithms or bot behavior and of course, deploy the algorithms to initiate the search procedure. When deployed, a radar is displayed to the user, which tracks the movements of the bot as well as the obstacles encountered and keeps a log of the nodes that have been visited. After a successful search, the radar highlights the shortest path calculated via the algorithm and runs the compiled video of the path.
Radar legend :
Red : Destination
Blue : Current TARS position
Dark Blue : Visited nodes
Green : Obstacles
Yellow : Nodes of the shortest path

CLIENT-SERVER ARCHITECTURE

In order to have a clean and robust design, and lower the use of computational resources of the Raspberry pi, we have employed a Client-Server architecture for TARS. It is equipped with a WI-FI adapter which allows it to communicate over the the network. The server and client side interfaces are designed from scratch using python socket programming. These communicate using a simple custom protocol for sending and receiving messages.

The main computational tasks of the algorithm are taken care of at the client, with only the instructions related to the bot's movement, obstacle detectoion and camera being communicated to the server. This keeps the workload off the 700MHz processor of the RPi, leaving the majority of resource hungry tasks to the more capable machine of the client.

Optical sensing

TARS has a 5 megapixel camera, which is used to take pictures at regular intervals, during the robot's navigation. These images are analyzed for a variety of purposes including identifying destination and compiling a video of the path.

Currently, We use Emmy award winning SSIM index algorithm to find similarities between two images that allows the bot to recognize destination and possibly other objects in near future. Other algorith that TARS uses is K-means clustering machine learning algorthim that allows it to quantize colors in the images into clusters, and then matching them with the target data.

Hardware and Thirdparty Software

  • Custom metal chassis
  • 2 Plastic tracked wheels: 67 x 26 mm

  • 2 DC Battery Operated motors
  • DNY power bank : 2 USB outputs
  • HC-SR04 ultrasonic sensor : 2 - 500 cm resolution
  • 5MP camera
  • Voltage divider circuit

Software dependencies

  • Language : Python
  • Server OS compatibility: Linux / Debian
  • Client OS compatibility: Linux, Windows
  • Pygame : Radar
  • easygui : GUI menus
  • scikit-learn : Image analysis
